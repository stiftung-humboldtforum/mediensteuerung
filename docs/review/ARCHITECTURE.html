Deployment-Modell

Host-Network überall: Alle Container teilen den Netzwerk-Namespace des Hosts. Kein Docker-Bridge-Networking. Konsequenz: jeder Service-Port direkt auf jedem Host-Interface erreichbar (Mongo :27017, Broker :8883, API :80/:443, FAC UDP :8080).
AspektKonfiguration
Container-Networknetwork_mode: host auf allen Python-Services
Source-Code-Distribution./backend/<svc>:/app bind-mounts → Hot-Patching via git pull (Dockerfile-Pins decorative)
TLS-Cert-Verteilung./backend/mkcert/certs/mqtt:/opt/tls in jeden Service (inkl. CA-Private-Key)
Service-Start-Orderdepends_on: condition: service_healthy auf broker + api
Privileged-Containernur manager (für ICMP-Raw-Socket via icmplib)
WatchdogHost-systemd-Unit (aktuell broken) restartet bei container-unhealthy
Cert-LifecycleOne-shot via docker compose -f docker-compose-certs.yml up — keine automatische Renewal
Build vs. Runbuild: + image: kombiniert; Image-Layer cached, Code aus bind-mount
