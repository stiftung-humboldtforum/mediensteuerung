Cross-Cutting-Themes (Architektur)

  1. 3× near-identische mqtt_client.py mit divergent Bug-State. Keiner re-subscribed bei Reconnect. Manager hat broken Offline-Queue (Race). Calendar/KNX byte-identisch stripped Versionen. → Shared Lib mit Reconnect/Resubscribe nötig.
  2. QoS-Asymmetrie: api/* qos=1, aber manager/*/calendar/* Events default qos=0 non-retained → Event-Loss bei transient drop.
  3. Broker-ACL fehlt → API-JWT/RBAC end-to-end moot. Jeder authn'd Cert kann manager/<fqdn>/<cmd> spoofen.
  4. Source-Bind-Mount auf allen 5 Services → Dockerfile-Pins decorative. Ein git pull swapped Prod ohne Rebuild.
  5. Dependency-Drift auf shared libs (paho-mqtt 1.x/2.x, aiomqtt 1.x vs gmqtt vs paho-raw, anyio 3.6.2 pinned in 3 vs unpinned in API, Python 3.11 vs 3.7.17).
  6. Implicit Topic-Schema — jeder Consumer string-splits Topics; Producer/Consumer-Mismatch bei fac/* (raw payload als Topic!) und event-payload-shape.
  7. TLS-Material überall verteilt: ca_key.pem + *.p12 in jeden Service bind-mounted obwohl von keinem referenziert.
  8. 5 Device-Control-Transports (MQTT/HTTP/SNMP/WebSocket/PJLink-TCP) bypassen MQTT-Control-Plane → kein zentral Audit/Auth/Retry.
  9. Logging divergent: print vs logging DEBUG/INFO/ERROR — keine shared Observability.
  10. Identity-Coupling probe.getfqdn() == NetBox.primary_ip.dns_name ohne Reconciliation → silent message-loss bei DNS-Drift.
  11. Manager privileged: true rein für ICMP-Raw-Socket — cap_add: [NET_RAW] würde reichen.
  12. Reproducible Builds nur auf Probe (uv-lock); alle 5 Backend-Services inline RUN pip ohne Hashes.
  13. Duplicate/overlapping MQTT-Subscriptions im Manager (probe/# global + probe/<fqdn>/+ per Device) verdoppeln Event-Dispatch.
Meta-Pattern: 5 Service-übergreifende Probleme (QoS-Mismatch, Duplicate-Subs, ACL-Gap, Reconnect-Story, Topic-Schema-Mismatch) lassen vermuten dass die Service-Konstellation als Ganzes nie End-to-End mit Broker-Restart getestet wurde.
